# 회의 보수적 VAD 재도입 설계

**날짜**: 2026-03-24
**상태**: 승인됨

## 목표

최근 변경에서 `/meeting` 기본 경로를 원본 오디오 중심으로 바꿨지만, 긴 파일 처리 시간이 길고 반복 노이즈가 보였다. 반면 기존 VAD 경로는 처리 시간은 짧아질 수 있지만 실제 발화까지 잘라먹는 문제가 있었다.

이번 변경의 목표는 다음과 같다.

1. VAD를 다시 기본 전략으로 사용할 수 있게 한다.
2. 다만 공격적 VAD가 아니라 `진짜 긴 무음만 제거`하는 보수적 VAD로 바꾼다.
3. 발화 손실을 최소화하는 것이 최우선이며, 무음이 일부 남는 것은 허용한다.

## 문제 정의

현재 `run_vad()`는 다음 설정을 사용한다.

- `min_silence_duration_ms = 3000`
- `speech_pad_ms = 300`

이 조합은 다음 문제를 일으킬 수 있다.

- 말과 말 사이의 짧은 멈춤도 경계로 인식될 수 있다.
- 발화 앞뒤 여유 구간이 짧아서 실제 음절 일부가 잘릴 수 있다.
- 결과적으로 STT 입력이 짧아지더라도 문장 누락/잘림이 생긴다.

## 선택한 방향

다음 세 가지 중 `침묵 기준 확대 + 패딩 확대`를 채택한다.

1. 침묵 기준만 확대
2. 침묵 기준 확대 + 패딩 확대
3. 침묵 기준 확대 + 패딩 확대 + 추가 병합 로직

선택 이유:

- 지금 문제는 “무음을 너무 많이 남긴다”가 아니라 “말을 잘라먹는다”이다.
- 따라서 `언제 자를지`를 늦추고, `잘라도 앞뒤를 넉넉히 남기는` 조합이 가장 직접적이다.
- 추가 병합 로직까지 넣으면 범위가 커지므로 이번에는 파라미터 조정으로 끝낸다.

## 변경 방향

### `pipeline.py`

- `run_vad()`는 유지한다.
- `VAD_MIN_SILENCE_DURATION_MS`를 현재 `3000ms`보다 크게 조정한다.
- `get_speech_timestamps()` 호출 시 `speech_pad_ms`를 현재 `300ms`보다 크게 조정한다.

추천 범위:

- `min_silence_duration_ms`: `5000ms ~ 6000ms`
- `speech_pad_ms`: `800ms ~ 1000ms`

구체 값은 테스트와 샘플 청취로 확정한다.

## 기대 효과

- 짧은 숨 고르기나 화자 전환 사이의 짧은 멈춤은 유지된다.
- 정말 긴 무음만 제거된다.
- VAD 결과 길이는 기존보다 더 길어지지만, 발화 손실은 줄어든다.
- STT 처리 시간은 원본 전체 처리보다는 짧고, 공격적 VAD보다는 길 수 있다.

## 오류 처리

- VAD 로직 자체는 유지하고, 실패 시 원본 오디오 fallback 정책도 유지한다.
- 새 파라미터가 너무 보수적이어서 거의 잘리지 않는 결과가 나와도 그것은 실패가 아니라 의도된 동작으로 본다.

## 검증 기준

다음 항목을 만족해야 한다.

1. 기존 VAD 결과보다 새 VAD 결과 길이가 더 길다.
2. 원본 오디오에 있는 발화가 VAD 결과에서 사라지는 체감이 줄어든다.
3. STT 결과에서 문장 잘림이 줄어든다.
4. 처리 시간이 늘어도 허용한다.

## 테스트 전략

자동 테스트:

- `VAD_MIN_SILENCE_DURATION_MS` 값 검증
- `get_speech_timestamps()`에 전달되는 `speech_pad_ms` 값 검증

수동 검증:

- 같은 회의 파일에 대해 기존 VAD 결과와 새 VAD 결과를 비교 청취
- 원본과 대조해서 실제 발화 손실이 줄었는지 확인
- STT 결과에서 문장 잘림이 줄었는지 확인

## 제외 항목

- 새로운 STT 엔진 도입
- VAD 후 추가 병합 알고리즘
- 반복 노이즈 후처리
- 2-pass 재인식

이번 범위는 `기존 VAD를 더 안전하게 만드는 것`에 집중한다.
