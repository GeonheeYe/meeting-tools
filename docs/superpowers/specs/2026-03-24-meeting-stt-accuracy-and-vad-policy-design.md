# 회의 STT 정확도 및 VAD 정책 개선 설계

**날짜**: 2026-03-24
**상태**: 승인됨

## 목표

현재 `/meeting` 파이프라인의 핵심 문제는 두 가지다.

1. VAD 전처리 과정에서 실제 발화가 잘려 STT 누락이 발생한다.
2. 도메인 용어가 STT 결과에서 자주 틀린다.

이번 변경의 목표는 다음과 같다.

1. 기본 경로에서 발화 손실을 막는다.
2. VAD를 기본 동작이 아니라 선택 옵션으로 내린다.
3. 긴 오디오에서도 누락/잘림을 줄이기 위해 STT 처리 방식을 개선한다.
4. 참고 문서 기반 용어 힌트를 강화해 도메인 용어 인식률을 높인다.

처리 시간 증가는 허용하며, 우선순위는 `누락 감소 > 용어 정확도 향상 > 기타 문장 품질`이다.

## 문제 정의

현재 흐름은 다음과 같다.

```text
원본 오디오
-> 필요 시 wav 변환
-> 기본적으로 VAD
-> Whisper STT
-> 선택적 화자 분리
```

이 구조의 문제는 다음과 같다.

- `*_vad.wav`에서 실제 발화가 잘려 나갈 수 있다.
- 긴 파일을 한 번에 STT할 경우 후반부 문맥 유지가 약해질 수 있다.
- 참고 문서 힌트는 들어가지만, 용어 우선순위와 별칭 정규화가 충분히 구조화되어 있지 않다.

## 선택한 방향

다음 세 가지 중 `기본 원본 STT + 청크 기반 처리 + 문서 기반 용어 강화`를 채택한다.

1. 기본 원본 STT + 보수적 후처리만 추가
2. 기본 원본 STT + 청크 기반 처리 + 문서 기반 용어 강화
3. 2-pass 재인식까지 포함한 공격적 정확도 개선

선택 이유:

- `누락/잘림`은 입력 손실 방지와 긴 오디오 처리 개선이 함께 필요하다.
- `도메인 용어 오인식`은 문서 기반 용어 구조화와 후처리 정규화가 효과적이다.
- 2-pass 재인식은 현재 범위 대비 복잡도와 비용이 크다.

## 변경 후 파이프라인

기본 흐름:

```text
원본 오디오
-> 필요 시 wav 변환
-> 문서 기반 용어 메타데이터 로드
-> 청크 기반 Whisper STT
-> 선택적 화자 분리
-> 문서 근거 기반 용어 정규화
```

옵션 흐름:

```text
원본 오디오
-> 필요 시 wav 변환
-> 명시적 VAD
-> 문서 기반 용어 메타데이터 로드
-> 청크 기반 Whisper STT
-> 선택적 화자 분리
-> 문서 근거 기반 용어 정규화
```

즉, VAD는 선택 기능이고, 정확도 개선의 기본 수단은 `원본 보존 + 청크 처리 + 용어 강화`다.

## 컴포넌트 설계

### `pipeline.py`

- 실행 정책의 중심을 맡는다.
- `use_vad: bool = False` 같은 인자를 추가해 기본값을 원본 오디오 STT로 둔다.
- VAD는 명시 옵션일 때만 실행한다.
- `context_loader` 결과를 받아 STT와 후처리에 전달한다.
- 결과 JSON에 다음 메타데이터를 저장한다.
  - `vad_applied`
  - `chunking_applied`
  - `source_audio_path`
  - `stt_audio_path`
  - `term_metadata_applied`

### `transcribe.py`

- STT 실행 책임을 가진다.
- `build_initial_prompt()`를 단순 키워드 나열이 아니라 아래 구조를 반영하도록 유지/보강한다.
  - 회의 주제
  - 기본 컨텍스트
  - 핵심 용어
  - 별칭 -> 정식명칭 표기 참고
- 긴 오디오를 안전하게 처리하기 위해 청크 기반 STT를 기본 경로로 사용한다.
- 청크 사이에는 짧은 오버랩을 두고, 병합 시 중복을 제거한다.

### `context_loader.py`

- 문서에서 추출한 용어 메타데이터를 신뢰 가능한 근거로 제공한다.
- 특히 아래 구조가 중요하다.
  - `priority_terms`
  - `alias_map`
- 문서에 실제 있는 용어만 STT 프롬프트와 후처리에 반영한다.

### `/meeting` 스킬

- 오케스트레이션만 담당한다.
- 기본 정책을 `원본 + 청크 STT`, `VAD 옵션`으로 문서화한다.
- 사용자가 VAD를 기본 동작으로 오해하지 않도록 실행 예시와 설명을 갱신한다.

## 청크 전략

청크 처리는 `누락/잘림` 감소를 위한 기본 전략이다.

- 긴 오디오를 일정 길이 기준으로 나눈다.
- 청크 경계에는 짧은 오버랩을 둔다.
- 병합 시 시간순 정렬 후 동일 문장/오버랩 중복을 제거한다.

목적은 실시간성이 아니라, Whisper가 긴 파일 후반부에서 문맥을 놓치지 않도록 하는 것이다.

## 용어 정확도 향상 전략

### 1. 구조화된 프롬프트

`initial_prompt`에 단순 키워드 나열 대신 다음 구조를 넣는다.

- 회의 주제
- 핵심 용어
- 별칭/약어와 정식명칭 매핑

### 2. 문서 근거 기반 정규화

후처리 교정은 생성이 아니라 정합성 보정으로 제한한다.

- 문서에 실제 존재하는 표현만 교정
- `alias_map`이 있는 경우 표기 흔들림을 정규화
- 문서에 없는 단어는 추측해서 삽입하지 않음

## 오류 처리

- 기본 경로에서는 VAD를 실행하지 않는다.
- `use_vad=True`일 때 VAD가 실패하면 원본 오디오로 fallback한다.
- 문서 로드 실패 시 용어 강화만 생략하고 기본 STT는 계속 진행한다.
- 1차 구현에서는 청크 실패 시 전체 실패를 허용하되, 실패 청크 정보는 로그에 남긴다.

## 검증 기준

다음 항목을 만족해야 한다.

1. 기본 실행은 원본 오디오를 사용한다.
2. 기본 실행은 청크 기반 STT를 사용한다.
3. VAD는 명시 옵션일 때만 적용된다.
4. VAD 실패 시 원본 오디오로 fallback한다.
5. 문서가 있을 때 `priority_terms`, `alias_map`가 프롬프트와 후처리에 반영된다.
6. 같은 파일 기준으로 기존 방식보다 문장 누락/잘림이 줄어든다.
7. 도메인 핵심 용어 표기 일치율이 좋아진다.

## 테스트 전략

자동 테스트:

- 기본 경로가 원본 오디오를 사용한다는 테스트
- VAD 옵션 경로와 fallback 테스트
- 청크 병합과 오버랩 중복 제거 테스트
- 프롬프트 구성 테스트
- 문서 기반 용어 정규화 테스트
- 결과 JSON 메타데이터 테스트

수동 검증:

- 같은 회의 파일에 대해 기존 경로와 새 기본 경로를 비교
- 원본 오디오를 직접 들으며 누락/잘림 감소 여부를 확인
- 문서 기반 핵심 용어가 더 잘 맞는지 확인

## 제외 항목

- 새로운 STT 엔진 도입
- 실시간 STT
- 화자 분리 품질 자체 개선
- 2-pass 재인식 기반 고비용 파이프라인

이번 범위는 현재 코드베이스 안에서 가장 큰 품질 개선 효과를 내는 변경에 집중한다.
