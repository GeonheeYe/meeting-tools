# 회의 STT용 오디오 보정 설계

**날짜**: 2026-03-24
**상태**: 승인됨

## 목표

현재 회의 녹음은 참가자마다 음량 차이가 크고, 작은 목소리는 STT에서 자주 씹히거나 오인식된다. 하지만 원본 오디오 자체는 보존해야 하므로, 저장된 녹음 파일을 직접 바꾸는 방식은 피해야 한다.

이번 변경의 목표는 다음과 같다.

1. 원본 오디오는 그대로 보존한다.
2. STT 직전에만 사용할 보정본 오디오를 별도로 생성한다.
3. 작은 목소리를 더 잘 들리게 하기 위해 `normalize + mild compression`을 적용한다.
4. 보정 실패 시에도 원본 경로로 계속 처리할 수 있게 한다.

## 선택한 방향

다음 세 가지 중 `원본 보존 + STT용 보정본 생성`을 채택한다.

1. 원본 파일 자체를 보정해서 저장
2. 원본은 그대로 두고 STT용 오디오만 보정
3. 녹음 단계에서 실시간 증폭

선택 이유:

- 원본 보존이 가능해야 비교와 디버깅이 쉽다.
- 녹음 단계 실시간 증폭은 장치/환경 영향을 크게 받고 검증이 어렵다.
- STT 직전 보정은 구현 범위가 작고 회귀 위험이 낮다.

## 변경 후 파이프라인

```text
원본 오디오
-> 필요 시 wav 변환
-> STT용 보정본 생성 (normalize + mild compression)
-> 선택적 VAD
-> STT
-> JSON 저장
```

원본 파일은 그대로 남고, 보정본은 임시 파일로만 사용한다.

## 컴포넌트 설계

### `pipeline.py`

- 오디오 변환 단계 다음에 `enhance_audio_for_stt()` 같은 함수를 추가한다.
- 입력은 wav 경로, 출력은 STT용 임시 wav 경로다.
- 이후 `run_vad()`와 `transcribe()`는 보정본을 기준으로 수행한다.
- 결과 JSON에 `audio_enhanced` 메타데이터를 추가한다.

### 보정 방식

1. **normalize**
- 전체 음량을 일정 기준으로 맞춘다.
- 너무 작은 입력은 적당히 끌어올린다.

2. **mild compression**
- 큰 소리는 조금 누르고, 작은 소리는 상대적으로 드러나게 만든다.
- 회의 녹음에서 음량 편차를 줄이는 용도다.

보정은 공격적이면 안 된다.
- 말소리가 찌그러지면 안 됨
- 잡음을 과하게 키우면 안 됨

## 오류 처리

- 보정이 실패하면 경고를 출력하고 원본/변환본으로 그대로 진행한다.
- 즉 보정은 품질 향상 단계이지, 파이프라인 필수 단계가 아니다.

## 검증 기준

다음 항목을 만족해야 한다.

1. 원본 오디오 파일은 변경되지 않는다.
2. 보정본 오디오는 STT에만 사용된다.
3. 작은 목소리가 기존보다 더 들리게 된다.
4. 큰 목소리가 과하게 왜곡되지 않는다.
5. STT에서 중간 씹힘이 줄어든다.

## 테스트 전략

자동 테스트:

- 보정 함수가 출력 파일을 생성하는지
- 보정 실패 시 원본으로 fallback하는지
- 결과 JSON에 `audio_enhanced`가 기록되는지

수동 검증:

- 같은 회의 파일로 원본 경로와 보정본 경로를 비교 청취
- 작은 목소리 구간이 실제로 더 잘 들리는지 확인
- STT 대화록에서 누락/씹힘이 줄었는지 확인

## 제외 항목

- 실시간 입력 증폭
- 강한 노이즈 억제
- 원본 녹음 파일 직접 수정
